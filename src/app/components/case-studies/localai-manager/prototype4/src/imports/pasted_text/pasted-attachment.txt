Build a fully interactive mobile AI chat application called "LocalAI" 
designed for iPhone 16 Pro (393×852pt). This is a portfolio-grade 
prototype — every state, every micro-interaction, and every transition 
must be fully realized.

─── VISUAL IDENTITY ───────────────────────────────────────

Typography:
- IBM Plex Mono — all UI labels, headers, badges, 
  status indicators, placeholders, metadata
- IBM Plex Sans (weight 300) — all conversation text, 
  body copy, descriptions

Color System (strict — no deviations):
- Background primary:   #0F0F0F
- Background surface:   #1A1A1A
- Background elevated:  #242424
- Text primary:         #E8E4DC
- Text secondary:       #9A9690
- Text subtle:          #5C5A57
- Accent:               #C8B98A
- Accent dim bg:        rgba(200,185,138,0.12)
- Accent border:        rgba(200,185,138,0.22)
- Borders:              #1E1E1E
- Deep bg:              #0A0A0A

No other colors. The single exception is #FF5555 for 
destructive actions (delete) only.

Device shell: iPhone 16 Pro with Dynamic Island.
Apply a very subtle noise/grain texture overlay across 
all screens at ~3% opacity.

── NAVIGATION & STATE PERSISTENCE ─────────────────────────

State that persists across sessions:
- Conversation history (stored locally)
- Selected conversation mode
- PC pairing status and device name
- Internet fetch toggle state

State that resets per session:
- Input field content
- Scroll position (starts at bottom)
- Any open sheets/dropdowns

Navigation transitions:
- Screen push (welcome → chat): 
  slide right, spring
- Sheet open (settings, QR, models): 
  slide up, spring  
- Menu open: slide right, spring
- Modal (action sheet): slide up, spring
- All closes: reverse of open

── DESIGN NOTES ────────────────────────────────────────────

The interface must feel familiar to 
ChatGPT/Claude users — same spatial 
logic, same input-at-bottom convention — 
but distinctly different through:
1. Monospace typography on all UI chrome
2. Amber accent instead of brand color
3. "local" / "on-device" language 
   replacing cloud terminology
4. Performance metrics visible on responses
5. The absence of avatar, account UI, 
   or any cloud iconography

The product communicates privacy not 
through copy but through what is absent:
no avatar, no account badge, 
no sync indicator pointing outward, 
no server imagery.

Every screen should render correctly 
inside iPhone 16 Pro shell with 
Dynamic Island at top.
Safe area insets must be respected:
top 59px, bottom 34px.
